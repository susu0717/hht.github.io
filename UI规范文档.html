数字认知游戏界面规范

Number Learning Game · 面向 UI 设计师 & 前端/客户端开发工程师

📱 目标平台：iOS / Android 平板 & 手机 👶 目标用户：2–6 岁幼儿（由家长陪同） 🎯 核心目标：数字 1–10 认知与顺序排列 🗓 版本：v1.0
Section 01

1界面概览与结构层级

🔍

界面解读：当前页面为「数字排序关卡」，屏幕顶部横排展示乱序数字卡片（6、1、3、2、5），中央区域有角色引导（蓝色小象吉祥物），右侧有第二个角色待命。玩法核心为：幼儿将数字拖拽/点击恢复为正确顺序。

🏗页面层级结构

  • Layer 0 背景层：纯色底色填充（珊瑚粉）
  • Layer 1 角色层：中央引导吉祥物（小象）、右侧待命角色
  • Layer 2 游戏元素层：可交互数字卡片组（顶部横排）
  • Layer 3 UI 控件层：返回按钮、进度条
  • Layer 4 反馈层：动效粒子、音效、正确/错误提示

📐布局模式

  • 整体布局：竖版满屏，横向居中对齐
  • 数字区：顶部 1/3 区域，Flexbox 横排等间距
  • 角色区：中部 1/2 区域，绝对定位
  • 控件区：顶部左上角返回键；底部进度条
  • 安全区：顶部 /底部 StatusBar 各留 safe area

Section 03

3角色（吉祥物）设计规范

🐘主引导角色 — 蓝色小象

  • 位置：屏幕中央，垂直居中偏下
  • 体色：薄荷蓝 #5BC8C4，耳朵浅粉 #FFB7C5
  • 造型：圆润卡通，大眼睛，友善表情
  • 手持道具：鼻子前端夹持一朵小花（增加亲近感）
  • 动画状态：待机时轻微上下浮动（2s loop）；答对时欢呼跳跃；答错时摇头安慰
  • 尺寸：约占屏幕宽度 45–50%

👾次要角色 — 绿色怪兽（右侧）

  • 位置：屏幕右下角，部分被裁切（暗示更多内容/下一关）
  • 体色：草绿 #6DD47E
  • 配件：头戴红色帽子，增加趣味性
  • 交互触发：点击该角色可能触发「查看下一关预告」或「角色介绍」弹层
  • 动画：待机时偶发探头动画（peek in/out）
  • 裁切意图：引导视线，暗示横向内容可滑动或下一关存在
⚠️

设计注意：所有角色需保持统一的圆润卡通风格，避免尖锐线条。角色表情需简洁明确（可用表情符号原型参考），确保 2 岁幼儿可识别。角色资源建议导出为 Lottie JSON 格式，便于开发实现骨骼动画与状态切换。

Section 04

4交互行为规范

🎮核心玩法交互流程

STEP 01

进入关卡

乱序数字以弹入动效从顶部出现，小象登场动画播放
STEP 02

引导提示

小象语音播报："请把数字按顺序排好！"（可选语音泡泡）
STEP 03

用户交互

拖拽或点击数字卡片，进行排序操作
STEP 04

实时反馈

每次放置后播放数字音效，小象给出表情反应
STEP 05

完成判定

全部正确 → 庆祝动效 + 音效；出现星星奖励 + 过关弹层

👆数字卡片交互方式

交互手势触发行为
单击/单触播放该数字读音（中文/英文可切换）
长按进入「拖拽模式」，卡片放大 1.15× 并跟随手指
拖拽中目标位置显示半透明占位框；其他卡片自动让位
释放（正确位置）吸附动效 + 闪光粒子 + 数字读音
释放（错误位置）弹回原位动效 + 轻柔错误提示音
双击数字字形展示小动画（如数字跳动）

📊数字卡片状态机

状态视觉表现
idle 待机正常显示，轻微呼吸动画（scale 1.0↔1.03）
hover 悬浮轻微放大 1.05×，投影加深
dragging 拖拽中放大 1.15×，旋转 ±5°，阴影强调，Z层置顶
correct 已放对绿色边框光晕，锁定（禁止再拖），✓ 小图标
wrong 放错红色闪烁一次，弹回原位，恢复 idle

🔊音效与语音反馈规范

触发时机
  • 点击数字 → 播报数字名称
  • 拖拽开始 → 轻柔拾取音
  • 放置正确 → 清脆奖励音
  • 放置错误 → 轻柔错误音
  • 全部完成 → 欢呼完成音乐（3–5秒）
音效设计要求
  • 所有音效时长 < 1.5秒（错误提示 < 0.8s）
  • 音调明亮、不刺耳（适合幼儿）
  • 语音录制采用女声/童声，亲切自然
  • 支持「静音模式」（家长设置）
语音多语言
  • 默认：普通话发音
  • 可选：粤语、英语
  • 语速：正常语速 × 0.85（清晰度优先）
  • 语音文件格式：.mp3 48kHz

5动效规范

✨动效时间 & 缓动曲线

动效场景时长缓动曲线说明
页面进场600msease-out数字依次弹入（stagger 80ms）
数字拖拽跟随实时线性零延迟响应，确保跟手感
卡片放置吸附200msspring (stiffness:400)弹性吸附效果
放置错误弹回350mscubic-bezier(.36,.07,.19,.97)抖动 + 弹回
正确完成粒子800msease-out星星/彩纸粒子爆发
吉祥物待机浮动2000ms loopease-in-outY轴 ±6dp，无限循环
答对角色欢呼1200msspring跳跃 + 挥手（Lottie 动画）
进度条更新400msease-in-out平滑增长
💡

开发建议：角色动画使用 Lottie（BodyMovin） 实现，iOS 使用 lottie-ios，Android 使用 lottie-android，React Native 使用 lottie-react-native。数字卡片交互动效可使用原生 Animated API 或 Reanimated 3 实现弹性效果。

Section 07

7无障碍 & 儿童适配要求

♿无障碍适配

  • 所有交互元素触控区域 ≥ 44×44dp
  • 色彩对比度：文字/图标与背景对比度 ≥ 4.5:1
  • 支持 VoiceOver / TalkBack，数字元素添加 accessibilityLabel
  • 动效提供「减少动效」模式适配（iOS Reduce Motion / Android）
  • 不依赖颜色作为唯一区分维度（加形状/位置辅助）

👶幼儿特殊适配

  • 防误触：返回/退出按钮需二次确认
  • 宽容点击区：幼儿触控精度低，扩大 20% 响应区域
  • 无广告区域：核心游戏区域内不出现任何广告或付费引导
  • 超时提示：静止 15s 后小象主动给出语音提示引导
  • 家长模式入口：隐藏在设置角落（三连击 logo 进入）
  • 内容分级：仅含适龄内容，符合 COPPA 规范

8开发交付清单

📦设计师交付物

  • 所有数字（1–10）完整 SVG / PNG 插图（@1x, @2x, @3x）
  • 角色（小象、绿色怪兽）各状态插图（待机/欢呼/摇头/跳跃）
  • Lottie JSON 动画文件（角色动画、粒子动画、完成庆祝动画）
  • Figma 源文件（含组件库、Color Style、Text Style）
  • 所有状态的界面标注文件（Figma Inspect 或 Zeplin）
  • 音效设计文档（或委托音效团队的需求说明）
  • 各屏幕尺寸适配说明（iPhone SE / iPhone Pro Max / iPad）

💻开发工程师实现要点

模块技术方案建议注意事项
拖拽排序iOS: UIKit DragInteraction / SwiftUI onDrag
Android: ItemTouchHelper / Jetpack Compose drag		需实现吸附逻辑、碰撞检测
角色动画Lottie（首选）/ Rive注意内存管理，及时释放不可见动画
